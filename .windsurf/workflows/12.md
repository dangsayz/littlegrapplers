## PATTERN INTELLIGENCE (NEW v2.0)

BORIS now maintains **Pattern Memory** - learned from recurring issues:

### Known Failure Patterns
1. **Enrollment Visibility Issues** - Parent pays but admin can't see enrollment
   - Root: Status mismatch (active vs pending_payment) or admin filters
   - Auto-fix: Sync status, check admin panel filters
   
2. **Payment/Enrollment Desync** - Stripe paid but enrollment not active
   - Root: Webhook failure or race condition
   - Auto-fix: Manual status sync, webhook replay
   
3. **Media Upload Failures** - "Object exceeded size" or "Failed to update"
   - Root: Supabase limits or missing DB columns
   - Auto-fix: Check limits, run migrations

4. **Age Validation Gaps** - System accepts but later rejects
   - Root: Frontend vs backend validation mismatch
   - Auto-fix: Align validation rules

### Pattern Detection Protocol
When `/12` is invoked, BORIS automatically:
1. Scans for known pattern matches in the request
2. Runs predictive diagnostics on related systems
3. Pre-loads likely solutions based on pattern history
4. Executes without waiting for human clarification
